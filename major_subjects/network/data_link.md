
# 1.데이터 링크의 역할
- 데이터 링크층에서 정보 전달은 단순히 0과 1을 나열하는 것이 아니라 '프레임'이라는 단위로 모아서 상대 기기에 전달한다.
# 2.데이터 링크의 기술
## MAC 주소
- MAC 주소는 데이터 링크에 연결되어 있는 노드를 식별하기 위해 사용된다.
- MAC 주소는 길이가 48비트로, 이 주소는 일반적인 네트워크 인터페이스 카드의 경우, ROM에 기록되어 있으며 동일한 MAC 주소를 가진 네트워크 인터페이스 카드는 전 세계에서 하나이다.
## 매체 공유형 네트워크
- 통신 매체를 여러 노드가 공유하는 네트워크이다.
- 매체 공유형 네트워크에서 우선권을 제어하는 장치로는 '컨텐션 방식'과 '토큰 패싱 방식'이 있다.
### 컨텐션 방식(CSMA)
- 데이터의 송신권을 경쟁을 통해 쟁탈하는 방식이다.
- 각 스테이션은 데이터를 송신하고 싶으면 먼저 보낸 사람이 이기는 방식으로 통신로를 사용하여 데이터를 송신한다.
- 충돌이 발생하지 않고 누구나 평등하게 송신권이 돌아오는 특징을 가진다.
### 토큰 패싱 방식
- 토큰이라는 패킷을 순환시킨 후 이 토큰으로 송신권을 제어한다.
- 토큰을 갖고 있는 스테이션만이 데이터를 보낼 수 있다.
## 매체 비공유형 네트워크
- 통신 매체를 공유하지 않고 전용하는 방식이다.
- 스테이션은 스위치라는 장치로 직접 연결되어 이 스위치가 프레임을 전송한다.
- 이 방식은 송수신하는 통신 매체를 공유하지 않기 때문에 대부분의 경우 전이중 통신이 된다.

# 3.이더넷(Ethernet)
- 현재 가장 보편화된 데이터 링크는 '이더넷'이다.
## 이더넷의 연결 형태
- 단말과 스위치 사이를 전용 케이블로 연결하여 이더넷 프로토콜로 통신하는 형태가 일반적이다.
## 이더넷의 종류
- 이더넷에는 통신 케이블이나 통신 속도가 다른 수많은 사양이 존재한다.
- 통신 속도는 같고 통신 케이블이 다른 경우, 각각 통신 매체를 변환할 수 있는 리피터나 허브등으로 연결할 수 있다.
- 통신 속도가 다른 경우, 속도 변환 기능을 가진 브리지나 스위칭 허브 또는 라우터를 필요로 한다.

# 4.무선통신
## 무선 통신의 종류
- 무선 통신은 해당 통신 거리에 따라 분류된다.
- IEEE802 위원회는 통신 거리에 따라 무선 PAN, 무선 LAN, 무선 MAN, 무선 RAN의 규격화를 진행 중이다.
- 무선 WAN의 경우, 휴대전화를 이용하여 통신을 들 수 있다.
## IEEE802.11
- IEEE802.11는 무선 LAN 프로토콜의 물리층과 데이터 링크층의 일부를 정의한 규격이다.
- MAC층에서 이더넷과 똑같은 MAC 주소를 사용한다.
- CSMA/CA라는 액세스 제어 방식을 채택하였다.
- 1Mbps 또는 2Mpbs의 통신 속도를 실현하는 규격이다.
## IEEE802.11b, IEEE802.11g
- IEEE802.11b와 IEEE802.11g는 2.4GHz대 전파를 이용하는 무선 LAN이다.
- 데이터 전송 속도는 각각 최대 11Mbps 및 54Mbps로, 통신할 수 있는 거리는 30 ~ 50m 정도이다.
- CSMA/CA라는 액세스 제어 방식을 채택하였다.
## IEEE802.11a
- 무선 LAN의 물리층으로, 5GHz대 주파수를 이용하여 최대 54Mbps까지의 전송 속도를 실현하는 규격이다.
## IEEE802.11n
- IEEE802.11a는 여러 개의 안테나를 동기시켜서 통신하는 MIMO라는 기술을 채택화하여 고속화르 실현한 것이다.
- 최대 300Mbps의 전송 속도르 이용할 수 있다.
## Bluetooth
- 블루투스는 2.4GHzㄷ 전파를 사용하는 통신 규격이다.
- 통신 가능한 거리의 종류에는 전파 강도에 따라 1m, 10m, 100m등이 있다.
- 통신 가능한 단말 수는 원칙적으로 최대 8대이다.
- 블루투스는 스마트폰, 키보드, 마우스같은 소형이고 전원 용량이 적은 기기를 대상으로 한다.

# 5.PPP(Point-to-Point Protocol)
## PPP란
- 1대1로 컴퓨터를 연결하기 위한 데이터 링크 프로토콜이다.
- PPP만으로는 통신할 수 없고, 물리층을 필요로 한다.
- PPP는 전화 회선, 전용 회선 등에서 이용된다.
## LCP와 NCP
- PPP는 데이터 통신을 시작하기 전에 PPP레벨에서 커넥션을 확립한다.
- LCP는 링크 설립, 검사, 설정, 감시 등의 기능을 하고, NCP는 다중 프로토콜 트래픽을 전송할 수 있는 기능을 한다.
### LCP
- 상위층에 의존하지 않는 프로토콜
- 커넥션의 확립 및 끊음, 패킷 길이의 설정, 인증 프로토콜 설정, 통신 품질의 감시를 할 것인지 등을 설정한다.
### NCP
- 상위층에 의존하는 프로토콜
- 상위층이 IP인 경우 NCP가 IPCP이다.
- IPCP에서는 IP 주소 설정, TCP/IP 헤더를 압축할 것인지 등의 정보를 주고받는다.
- IPCP : PPP 프로토콜에서 IP 에 연관되는 네트워크 제어 프로토콜(NCP)
## PPP의 프레임 포맷
<img width="450" alt="image" src="https://user-images.githubusercontent.com/110087065/211174952-6de3cd94-9d1b-4b5e-a412-411a9692577c.png">
– Flag (1 Byte) : PPP 프레임의 시작 지점을 나타낸다. 항상 ‘01111110’으로 고정된다.<br/>
– Address (1 Byte) : HDLC에서 프레임 목적지 주소를 의미한다. 하지만 PPP는 두 장비간 연결이므로 의미 없는 필드다. 그래서 브로드캐스트 주소 ‘11111111’의 값으로 고정된다.<br/>
– Control (1 Byte) : HDLC에서 여러 제어 목적에 사용되나, PPP에서는 ‘00000011’으로 고정된다.<br/>
– Protocol (2 Byte) : 프레임 내 Information에 캡슐화된 데이터그램의 프로토콜을 식별한다. 아래에 좀 더 자세히 살펴보도록 하겠다.<br/>
– Information (가변) : 데이터 또는 제어 정보를 포함한다.<br/>
– Padding (가변) : PPP 프레임 크기를 맞추기 위해 더미 바이트를 추가할 수 있다.<br/>
– FCS (2 or 4 Byte) : 전송 중 에러로부터 프레임을 보호하기 위한 방법이다. CRC checksum과 비슷하다.<br/>
– Flag (1 Byte) : PPP 프레임의 끝 지점을 나타낸다. 항상 ‘01111110’으로 고정된다.<br/>

# 6.그 밖의 데이터 링크
## ATM (Asynchronous Transfer Mode,비동기전달모드)
- ATM은 데이터를 헤더 5옥텟 + 데이터 48옥텟을 세트 단위로 처리하는 데이터 링크이다.
- 회선의 점유 시간을 짧게 함으로써 대용량 데이터를 효율적으로 전송 가능하다.
- 커넥션 지향 데이터 링크이므로 통신을 시작하기 전에 통신 회선을 설정해야한다.
- ATM은 송신권의 제어는 없고, 필요할 때 필요한 만큼 데이터를 송신할 수 있다.
## POS (Packet Over SONET/SDH)
- POS는 디지털 신호를 광섬유로 주고받기 위한 물리층의 규격인 SDH(동기 디지털 계층)상에서 패킷 통신을 하기 위한 프로토콜이다.
- 신뢰성 높은 광전송 네트워크로 사용되고 있다.
## FDDI (Fiber Distributed Digital Interface)
- FDDI는 광섬유나 트위스트 페어 케이블을 사용하여 100Mbps의 전송 속도를 실현할 수 있기 때문에 네트워크의 백본이나 컴퓨터 간을 고속으로 연결하기 위해 이용된다.
- 토큰 패싱 방식을 채택한다.
## 토큰링
- IBM이 개발한 토큰 패싱 방식의 LAN으로 4Mbps 또는 16Mbps의 데이터 전송 속도를 실현한다.
## 100VG-AnyLAN
- UTP 케이블에서 100Mbps 속도를 실현한다.
- 디맨드 우선 순위 방식을 채택한다.
- 스위치가 송신권을 제공한다.
## 파이버 채널
- 고속 데이터 채널을 실현하는 데이터 링크이다.
- 주변 기기를 연결하는 버스에 가까운 구조로 되어 있기 때문에 133Mpbs ~ 4Gbps의 데이터 전송 속도를 실현한다.
## HIPPI
- 슈퍼 컴퓨터끼리 연결에 사용된다.
- 800Mbps 또는 1.6Gbps의 데이터 전송 속도를 실현한다.
## IEEE 13942
- 가정용 LAN에서 사용되는 데이터 링크이다.
- 100~800Mbps 이상의 데이터 전송 속도를 실현한다.
## HDMI
- 케이블 하나로 고품질의 영상과 음성을 전송할 수 있느 규격이다.
- 컴퓨터, 태블릿 PC, 디스플레이 등에 사용된다.
## iSCSI
- PC등에 하드디스크를 연결하기 위한 표준 규격인 SCSI를 TCP/IP 네트워크 상에서 이용하는 규격이다.
- SCSI 명령과 데이터를 IP 패킷에 포함시켜 데이터를 송수신한다.
## InfiniBand
- 하이엔드 서버용으로 만들어지 초고속 인터페이스이다.
- 고속, 고신뢰성, 저지연 등의 특징을 가진다.
- 여러 개의 케이블을 하나로 묶어 이용하여 몇 백 Gbps의 전송 속도를 실현 가능하다.
## DOCSIS
- 케이블 TV에서 데이터 통신을 하기 위한 업계 표준 규격이다.
- 동축 케이블에 케이블 모뎀을 연결하고 이더넷과 교환하기 위한 사양을 표준화한다.
## 고속 PLC
- 종래의 전력선을 이용하여 몇십 ~ 200 Mbps의 전송 속도를 실현한다.
- 전력선으 통신에 사용하여 LAN을 새로 배선하지 않아도 된다.
- 전파 누수의 가능성이 있다.


# 7.공중 액세스망
## 아날로그 전화 회선
- 고정 전화 회선을 이용하여 통신한다.
- 컴퓨터를 전화 회선으로 연결하기 위해 디지털 신호와 아날로그 신호를 교환해주는 모뎀이 필요하다.
- 통신 속도가 매우 느리다.
## 이동 통신 서비스
- 서비스 이용 가능한 범위에 있으면 어디서든지 네트워크에 연결할 수 있다.
- PHS와 휴대전화를 이용할 수 있다.
- Mobile WiMAX, LTE같은 기술로 몇십 Mbps 정도의 데이터 통신을 할 수 있다.
## ADSL
- 아날로그 전화 회선을 확장, 이용하는 서비스이다.
- 전화기와 전화국 교환기 사이의 회선을 이용한다.
- 스플리터라는 분배기를 설치하여 음성 주파수와 데이터 통신용 주파수를 혼합 및 분리한다.
## FTTH
- 고속의 광섬유를 사용자의 집이나 회사 건물 안에 직접 끌어들이는 방법이다.
- ONU라는 장치를 이용하여 빛을 전기 신호로 변환한 후 컴퓨터나 라우터에 연결한다.
## 케이블 TV
- 케이블을 사용하여 방송하는 서비스이다.
- 케이블을 사용하여 전파의 수신 상태가 나빠지는 경우를 보완할 수 있다.
## 전용 회선
- 전용 회선 서비스의 저가화, 광대역화, 다양화로 '전용선 서비스'가 제공된다.
- 국내 일반 전용 회선은 필요한 만큼 1시간 단위로 고품질의 영상을 전송할 수 있는 서비스이다.
- Kornet Premium은 네트워크에서 고객에게 필요한 부가서비스를 제공하는 이더넷 서비스이다.
- Kornet Express는 초고속 백본망으로 연결되는 Metro Ethernet 방식의 인터넷 전용 회선이다.
- Kornet Hotline은 초고속 인터넷망과 전용 회선으로 구성된 고품질 인테넛 전용 회선이다.
## VPN(Virtual Private Network)
- 떨어진 지역을 이어주는 VPN 서비스의 예로는 IP-VAP와 광역 이더넷을 들 수 있다.
### IP-VPN
  - 'IP 네트워크로 VPN을 구축하는 방법'과 'IPsec을 사용하여 VPN을 실현하는 방법'이 존재한다.
  - MPLS 기술을 사용하여 VPN을 구축하는 서비스를 통신 사업자 제공한다.
    - MPLS : 라벨이라는 정보를 IP 패킷에 추가하여 통신을 제어한다. 라벨을 고객별로 다르게 설정하여 MPLS망을 통과할 때 라벨별로 수신처를 판단한다.
  - IPsec에 의해 VPN 상에서 통신 시 IP 패킷의 인증, 암호화를 하여 인터넷상에서 닫힌 네트워크를 구축한다.
### 광역 이더넷
  - 통신 사업자가 제공하는 떨어진 지역을 잇는 인터넷 연결 서비스이다.
  - 광역 이더넷은 데이터 링크층인 이더넷을 사용한 VLAN을 이용한다.
  - 이더넷을 그대로 사용하므로 TCP/IP 이외의 프로토콜도 사용 가능하다.
  - 똑같은 VLAN을 지정하면 어디서든 동일한 네트워크에 연겨 가능하다.
  - 데이터 링크층을 이용하기 때문에 불필요한 패킷을 보내지 않도록 운용하여야 한다.
## 공중 무선 LAN
- Wi-Fi를 이용한 서비스이다.
- 핫스팟이라는 전파 수신 가능 영역을 이용 장소에 설치하고 무선 LAN 인터페이스르 가진 기기르 연결한다.
