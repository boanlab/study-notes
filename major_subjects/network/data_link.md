
# 1.데이터 링크의 역할
- 데이터 링크층에서 정보 전달은 단순히 0과 1을 나열하는 것이 아니라 '프레임'이라는 단위로 모아서 상대 기기에 전달한다.
# 2.데이터 링크의 기술
## MAC 주소
- MAC 주소는 데이터 링크에 연결되어 있는 노드를 식별하기 위해 사용된다.
- MAC 주소는 길이가 48비트로, 이 주소는 일반적인 네트워크 인터페이스 카드의 경우, ROM에 기록되어 있으며 동일한 MAC 주소를 가진 네트워크 인터페이스 카드는 전 세계에서 하나이다.
## 매체 공유형 네트워크
- 통신 매체를 여러 노드가 공유하는 네트워크이다.
- 매체 공유형 네트워크에서 우선권을 제어하는 장치로는 '컨텐션 방식'과 '토큰 패싱 방식'이 있다.
### 컨텐션 방식(CSMA)
- 데이터의 송신권을 경쟁을 통해 쟁탈하는 방식이다.
- 각 스테이션은 데이터를 송신하고 싶으면 먼저 보낸 사람이 이기는 방식으로 통신로를 사용하여 데이터를 송신한다.
- 충돌이 발생하지 않고 누구나 평등하게 송신권이 돌아오는 특징을 가진다.
### 토큰 패싱 방식
- 토큰이라는 패킷을 순환시킨 후 이 토큰으로 송신권을 제어한다.
- 토큰을 갖고 있는 스테이션만이 데이터를 보낼 수 있다.
## 매체 비공유형 네트워크
- 통신 매체를 공유하지 않고 전용하는 방식이다.
- 스테이션은 스위치라는 장치로 직접 연결되어 이 스위치가 프레임을 전송한다.
- 이 방식은 송수신하는 통신 매체를 공유하지 않기 때문에 대부분의 경우 전이중 통신이 된다.

# 3.이더넷(Ethernet)
- 현재 가장 보편화된 데이터 링크는 '이더넷'이다.
## 이더넷의 연결 형태
- 단말과 스위치 사이를 전용 케이블로 연결하여 이더넷 프로토콜로 통신하는 형태가 일반적이다.
## 이더넷의 종류
- 이더넷에는 통신 케이블이나 통신 속도가 다른 수많은 사양이 존재한다.
- 통신 속도는 같고 통신 케이블이 다른 경우, 각각 통신 매체를 변환할 수 있는 리피터나 허브등으로 연결할 수 있다.
- 통신 속도가 다른 경우, 속도 변환 기능을 가진 브리지나 스위칭 허브 또는 라우터를 필요로 한다.

- 프레임 본체의 수신처 MAC 주소 필드 6옥텟, 송신처 MAC 주소 필드 6옥텟, 데이터 부분에서 전송하고 있는 상위층 프로토콜 종류를 나타내는 필드 2옥텟이 존재한다.

- 프레임의 끝에는 FCS(Frame Check Sequence)라는 4옥텟의 필드가 존재한다.

# 4.무선통신
## 무선 통신의 종류
- 무선 통신은 해당 통신 거리에 따라 분류된다.
- IEEE802 위원회는 통신 거리에 따라 무선 PAN, 무선 LAN, 무선 MAN, 무선 RAN의 규격화를 진행 중이다.
- 무선 WAN의 경우, 휴대전화를 이용하여 통신을 들 수 있다.
## IEEE802.11
- IEEE802.11는 무선 LAN 프로토콜의 물리층과 데이터 링크층의 일부를 정의한 규격이다.
- MAC층에서 이더넷과 똑같은 MAC 주소를 사용한다.
- CSMA/CA라는 액세스 제어 방식을 채택하였다.
- 1Mbps 또는 2Mpbs의 통신 속도를 실현하는 규격이다.
## IEEE802.11b, IEEE802.11g
- IEEE802.11b와 IEEE802.11g는 2.4GHz대 전파를 이용하는 무선 LAN이다.
- 데이터 전송 속도는 각각 최대 11Mbps 및 54Mbps로, 통신할 수 있는 거리는 30 ~ 50m 정도이다.
- CSMA/CA라는 액세스 제어 방식을 채택하였다.
## IEEE802.11a
- 무선 LAN의 물리층으로, 5GHz대 주파수를 이용하여 최대 54Mbps까지의 전송 속도를 실현하는 규격이다.
## IEEE802.11n
- IEEE802.11a는 여러 개의 안테나를 동기시켜서 통신하는 MIMO라는 기술을 채택화하여 고속화르 실현한 것이다.
- 최대 300Mbps의 전송 속도르 이용할 수 있다.
## Bluetooth
- 블루투스는 2.4GHzㄷ 전파를 사용하는 통신 규격이다.
- 통신 가능한 거리의 종류에는 전파 강도에 따라 1m, 10m, 100m등이 있다.
- 통신 가능한 단말 수는 원칙적으로 최대 8대이다.
- 블루투스는 스마트폰, 키보드, 마우스같은 소형이고 전원 용량이 적은 기기를 대상으로 한다.

# 5.PPP(Point-to-Point Protocol)
## PPP란
- 1대1로 컴퓨터를 연결하기 위한 데이터 링크 프로토콜이다.
- PPP만으로는 통신할 수 없고, 물리층을 필요로 한다.
- PPP는 전화 회선, 전용 회선 등에서 이용된다.
## LCP와 NCP
- PPP는 데이터 통신을 시작하기 전에 PPP레벨에서 커넥션을 확립한다.
### LCP
- 상위층에 의존하지 않는 프로토콜
- 커넥션의 확립 및 끊음, 패킷 길이의 설정, 인증 프로토콜 설정, 통신 품질의 감시를 할 것인지 등을 설정한다.
### NCP
- 상위층에 의존하는 프로토콜
- 상위층이 IP인 경우 NCP가 IPCP이다.
- IPCP에서는 IP 주소 설정, TCP/IP 헤더를 압축할 것인지 등의 정보를 주고받는다.
## PPP의 프레이 포맷
<img width="450" alt="image" src="https://user-images.githubusercontent.com/110087065/211174952-6de3cd94-9d1b-4b5e-a412-411a9692577c.png">
– Flag (1 Byte) : PPP 프레임의 시작 지점을 나타낸다. 항상 ‘01111110’으로 고정된다.<br/>
– Address (1 Byte) : HDLC에서 프레임 목적지 주소를 의미한다. 하지만 PPP는 두 장비간 연결이므로 의미 없는 필드다. 그래서 브로드캐스트 주소 ‘11111111’의 값으로 고정된다.<br/>
– Control (1 Byte) : HDLC에서 여러 제어 목적에 사용되나, PPP에서는 ‘00000011’으로 고정된다.<br/>
– Protocol (2 Byte) : 프레임 내 Information에 캡슐화된 데이터그램의 프로토콜을 식별한다. 아래에 좀 더 자세히 살펴보도록 하겠다.<br/>
– Information (가변) : 데이터 또는 제어 정보를 포함한다.<br/>
– Padding (가변) : PPP 프레임 크기를 맞추기 위해 더미 바이트를 추가할 수 있다.<br/>
– FCS (2 or 4 Byte) : 전송 중 에러로부터 프레임을 보호하기 위한 방법이다. CRC checksum과 비슷하다.<br/>
– Flag (1 Byte) : PPP 프레임의 끝 지점을 나타낸다. 항상 ‘01111110’으로 고정된다.<br/>

# 6.그 밖의 데이터 링크
## ATM
- ATM은 데이터를 헤더 5옥텟 + 데이터 48옥텟을 세트 단위로 처리하는 데이터 링크이다.
- 회선의 점유 시간을 짧게 함으로써 대용량 데이터를 효율적으로 전송 가능하다.
- 커넥션 지향 데이터 링크이므로 통신을 시작하기 전에 통신 회선을 설정해야한다.
- ATM은 송신권의 제어는 없고, 필요할 때 필요한 만큼 데이터를 송신할 수 있다.
